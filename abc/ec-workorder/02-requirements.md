# EC InfoTech 工单系统需求规格

## 1. 项目背景

EC InfoTech 当前工单、维修、报价、采购、报销、归档存在纸质表单、WhatsApp、Excel、文件夹和旧系统并行的情况。一线师傅认为系统复杂，现场资料录入不完整；管理层看不到实时状态；照片、员工纸、报价和发票归档分散；大量图片上传导致旧系统卡顿和容量压力。来源：M1、M2、M4、M5、M6、I4、I8、I16。

## 2. 项目目标

| 目标 | 一期判断 | 来源 |
|---|---|---|
| 替代纸质Daily Service Call Form，电子化记录来电/WhatsApp工单 | 必做 | M6、I4、I14 |
| 建立PM/CM分类、区域/团队分派、移动端执行闭环 | 必做 | M4、M5、M6、I1-I3、I20 |
| 师傅现场拍照、签到、填写处理结果，形成可追溯服务记录 | 必做 | M2、M4、M6、I11、I12、I16 |
| 管理层查看基础工单状态和团队工作量 | P0必做；Outstanding原因看板为P1可选 | M5、I5、I6、I18 |
| 报销、采购、报价与工单编号关联 | 报销为P1可选；采购/报价为P2字段预留；完整流程二期增强 | M4、M5、M6、I21-I31 |
| AI识别、客户自助查询、WhatsApp自动抓取、语音转写 | 三期规划 | M1、M5、M6 |

## 3. 用户角色

| 角色 | 核心职责 | 一期权限 |
|---|---|---|
| CS | 接听电话/WhatsApp、录入来电、判断是否即时解决或生成工单。 | 新建/编辑本人创建的未派工单，查看工单状态，上传来电附件。 |
| CR | 处理客户投诉/客户关系跟进，必要时发起或接收投诉类工单。 | 查看客户相关工单，创建/转派投诉工单，添加跟进记录。 |
| Team Leader | 接收区域/团队工单，分派师傅，审核拒单原因，跟踪Outstanding。 | 查看团队工单，分派/改派，确认完成或退回补充。 |
| 师傅/外勤 | 移动端接单/拒单、到场签到、拍照、填写处理结果、提交完成。 | 只看本人/本组可接工单和已分派工单，不进入复杂后台。 |
| 报价人员 | 对需要报价的工单补充报价资料。 | 一期可记录“需报价/报价编号/负责人/状态”，完整报价二期。 |
| 采购人员 | 对需要材料的工单记录采购需求、PO、预计/实际到货、ML。 | 一期维护采购关联字段和状态，不做库存自动校验。 |
| 财务/会计 | 根据工单、报价、员工纸/照片、报销记录进行发票/归档。 | 查看已完成工单和报销/采购/报价关联记录，导出。 |
| 管理层 | 查看状态、团队工作量、Outstanding原因、趋势。 | 看板和导出权限，不直接修改现场记录。 |
| 系统管理员 | 用户、团队、区域、权限、编号规则、客户档案维护。 | 全量配置和操作日志查看。 |

## 4. 一期 MVP 范围

一期按 P0/P1/P2 分级确认。P0 是必须上线的最小闭环；P1 是一期可选增强，需看预算、POC结果和客户确认；P2 仅预留字段，不承诺流程交付。

| 模块 | 必做内容 | 来源 |
|---|---|---|

| 区域/Team Leader推荐 | 根据客户地址/区域码推荐Team Leader；区域码兼容1P/2P/4P/5P/6P/7P。 | M4、M6、I19 |

## 8. 核心业务流程

### 8.1 来电/WhatsApp创建工单

1. CS录入客户、联系人、电话、来电时间、Call Code、Call Content、区域、地址、附件。
2. 若简单问题可电话解决，状态直接为已完成/已归档。
3. 若需现场处理，生成PM或CM工单并进入待处理。

来源：M6、I4、I14。

### 8.2 PM/CM工单分类

| 类型 | 定义 | 派单逻辑 |
|---|---|---|
| PM | 常规维修/保养类任务。 | 按区域、排程、Team Leader分派。 |
| CM | 特别任务，无固定执行人。 | CS/CR指定，按技能和空闲情况临时指派；拒单后重新分派。 |

来源：M6、I3。

### 8.3 区域/团队分派

系统根据客户地址/屋苑/区域码推荐Team Leader；支持手动调整、多团队同时指派。区域码需兼容现有1P/2P/4P/5P/6P/7P体系。来源：M4、M6、I19。

### 8.4 工单池主动接单（P1可选）

未指定或积压工单可进入工单池；师傅/团队可申请接单；Leader确认后成为已接单。来源：M5。

### 8.5 师傅接单/拒绝

师傅移动端只显示必要信息：地点、联系人、问题、要求到场时间、是否可拍照、附件。可点击接单或拒绝；拒绝必须选择原因，如正在执行其他PM任务。来源：M5、M6。

### 8.6 到场签到/拍照/处理结果

到场时记录时间、位置/选点、可选照片；处理后上传before/after照片、处理结果、是否需报价/材料/客户确认。不可拍照客户不强制上传照片，但需填写原因。来源：M2、M4、M5、I11、I12、I16。

### 8.7 工单完成/主管确认

师傅提交完成后进入待主管确认。若客户确认/照片/备注完整，Leader确认已完成；若信息不足则退回补充。会议也提到“建议简化主管审批”，因此是否强制主管确认需客户最终确认。来源：M4、M5。

### 8.8 需要报价（P2字段预留）

工单标记为等待报价，记录报价负责人、报价编号、报价状态。完整报价单、版本控制、客户确认放二期。来源：M4、M5、I21-I28。

### 8.9 需要采购（P2字段预留）

工单标记为等待材料，记录物料描述、照片、PO编号、预计/实际到货、ML编号。库存由采购人工确认。来源：M4、M6、I29、I31。

### 8.10 报销与工单关联（P1可选）

报销记录可选择一个或多个已完成工单，上传票据/手写报销单；一期先做关联和附件，AI识别作为POC或二期试点。来源：M4、M5、M6。

## 9. 工单状态设计

| 状态 | 说明 | 进入条件 | 退出条件 |
|---|---|---|---|
| 待处理 | CS已创建，未分派/未接单。 | 新建维修工单。 | 派单、取消、已解决。 |
| 已接单 | 师傅/团队确认接受。 | 指派后接单或工单池接单确认。 | 到场/开始处理。 |
| 处理中 | 已开始现场/后续处理。 | 到场签到、检查或跟进。 | 完成、等待报价、等待材料、等待客户确认、取消。 |
| 等待报价 | 需报价后才能继续。 | 师傅或Leader标记需报价。 | 报价完成/客户确认/取消。 |
| 等待客户确认 | 已报价或完成但需客户确认。 | 报价发出、Job Completion待签。 | 客户确认/取消/继续处理。 |
| 等待材料 | 需采购/到货。 | 标记需采购或PO创建。 | 材料到货/继续处理/取消。 |
| 待主管确认 | 师傅提交完成，待Leader核查。 | 提交完成。 | 已完成或退回处理中。 |
| 已完成 | 现场或后续处理完成。 | Leader确认或简化流程下师傅完成。 | 归档。 |
| 已取消 | 工单无需处理或重复/错误。 | CS/Leader取消。 | 归档。 |
| 已归档 | 财务/报告/照片完成归档。 | 已完成或已取消并归档。 | 关闭。 |

## 10. “处理中”子状态设计

| 子状态 | 用途 |
|---|---|
| 已到场 | 已签到/到达现场。 |
| 检查中 | 正在判断故障。 |
| 未能即时维修 | 当场不能解决，需要后续报价、采购或排期。 |
| 填写报价资料 | 已采集报价所需资料和照片。 |
| 等待材料 | 已确认需物料，等待采购/到货。 |
| 施工中 | 现场作业进行中。 |
| 等待客户资料/确认 | 缺客户确认、签名、进入许可或资料。 |
| 已提交待核查 | 师傅提交完成，待主管确认。 |

来源：M5。

## 11. 编号规则

| 编号 | 规则 | 示例/备注 | 来源 |
|---|---|---|---|
| 普通小区类工单 | 日期+当日序号 | 20260326-1 | M6 |
| 政府合约类工单 | 合约/前缀字母+ECA+日期+序号 | ECA20260326-1；具体前缀需确认 | M6 |

| S/R编号 | Service Report/报价关联 | 报价样张含S/R No. | I21-I28 |

## 12. 客户/地址/区域主数据

必填字段建议：客户名称（繁体中文）、英文别名、客户类型、联系人、电话、地址、屋苑/楼宇、区域、区域码、Team Leader、是否可拍照、特殊进入要求、合约/项目关系。项目名称必须优先使用繁体中文，并补充英文别名支持检索。来源：M4、M5、I16。

## 13. 拍照与不可拍照客户配置

客户档案增加“是否可拍照/影像限制”字段。可拍照时，现场至少上传处理前/处理后或关键证据照片；不可拍照时，系统取消照片必填，师傅填写文字说明和替代证明。来源：M5、I26。

## 14. 查询、筛选、模糊搜索

支持按创建时间、状态、负责人、团队、区域、客户、项目编号、PO、S/R、关键词筛选。模糊搜索需支持中文/英文各自匹配；跨语言同义词如“电池/battery”一期不承诺。来源：M4、M5。

## 15. 报表与管理看板

P0看板：待处理、已接单、处理中、已完成、已取消、各区域待办、各团队基础工作量、照片缺失/不可拍照统计。

P1看板：Outstanding原因分类、超时清单、等待报价/等待材料/等待客户确认细分。来源：M5、I5、I6、I18。

## 16. 权限与操作日志

所有用户独立ID，不允许共用账号。记录创建、派单、接单、拒单、状态变更、照片上传、退回、确认、取消、导出等操作。来源：M5。

## 17. 通知与超时提醒

## 18. 数据导入导出

一期P0需要导入客户/地址/区域/团队/用户基础数据和历史未完成工单清单。导出支持工单列表、照片/附件索引。Outstanding Report和报销关联清单属于P1可选。来源：M4、M6、I18、I31。

| 编号 | 验收项 |
