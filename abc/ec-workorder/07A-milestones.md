# 一期MVP 6周里程碑

本排期以“P0必须上线”为基准，P1可选项需在第1周确认是否纳入。若客户新增未确认需求，进入变更池，不直接加入一期。

## 第1周：需求确认 + 字段确认 + HTML原型

| 项 | 内容 |
|---|---|
| 工作内容 | 确认P0/P1/P2范围；确认PM/CM定义、编号规则、区域/Team Leader规则、不可拍照客户；制作HTML原型。 |
| 交付物 | 范围确认稿、字段确认表、HTML原型、待确认问题清单。 |
| 客户配合事项 | 提供真实Daily Service Call、Service Report、区域/团队表、客户/地址样表；安排CS、Leader、师傅评审原型。 |
| 验收标准 | 客户确认P0范围；原型覆盖CS建单、Leader派单、师傅移动端、工单详情、看板。 |

## 第2周：数据模型 + 基础资料导入 + 建单流程

| 项 | 内容 |
|---|---|
| 工作内容 | 搭建工单主表、客户/地址/团队配置、附件表、流转记录表；导入试点客户/地址/人员；实现CS手工建单和自动编号。 |
| 交付物 | 数据表、导入模板、试点基础数据、建单页面。 |
| 客户配合事项 | 提供试点数据并确认清洗结果；确认政府前缀和普通工单编号规则。 |
| 验收标准 | CS可创建工单；系统自动编号；地址可带出区域和推荐Leader。 |

## 第3周：派单工作台 + 师傅移动端

| 项 | 内容 |
|---|---|
| 工作内容 | 实现Leader派单/改派、拒单处理、师傅移动端我的待办、接单/拒绝、工单详情。 |
| 交付物 | Leader派单工作台、师傅移动端P0页面、基础权限。 |
| 客户配合事项 | 安排试点Leader和师傅测试；确认拒单原因枚举。 |
| 验收标准 | Leader可把工单派给师傅；师傅可接单/拒绝；拒单原因可追溯。 |

## 第4周：拍照上传 + 状态流转 + 通知提醒

| 项 | 内容 |
|---|---|
| 工作内容 | 实现到场记录、拍照上传、不可拍照豁免、处理结果提交、状态按钮、基础通知。 |
| 交付物 | 现场执行流程、照片附件分类、状态流转规则、通知模板。 |
| 客户配合事项 | 提供真实手机和现场网络测试；确认是否强制Leader完成确认；确认通知对象。 |
| 验收标准 | 师傅可从接单走到提交结果；照片或不可拍照原因完整；状态变化有日志。 |

## 第5周：查询看板 + 权限日志 + UAT

| 项 | 内容 |
|---|---|
| 工作内容 | 实现工单列表筛选、基础管理看板、操作日志、权限验证；组织UAT。 |
| 交付物 | 工单列表、看板、权限矩阵、UAT问题清单。 |
| 客户配合事项 | CS、Leader、师傅、管理层参与UAT；确认P1是否上线或后置。 |
| 验收标准 | 可按时间、状态、区域、客户、负责人查询；师傅看不到非授权工单；管理层可看到基础状态分布。 |

## 第6周：试点上线 + 反馈修复 + 验收

| 项 | 内容 |
|---|---|
| 工作内容 | 试点上线、处理UAT反馈、修复高优先级问题、准备验收材料和培训。 |
| 交付物 | 试点环境、用户操作说明、验收清单、上线问题跟踪表。 |
| 客户配合事项 | 指定试点团队真实使用；提供每日反馈；确认验收结果和二期需求池。 |
| 验收标准 | P0流程可闭环；试点工单可从建单到完成；照片上传成功率达到POC标准；客户签署一期验收或试点通过确认。 |
