# 一期MVP范围

## MVP结论

一期MVP可行，但必须严格聚焦“工单闭环”：CS手工建单、自动编号、PM/CM分类、区域/Team Leader推荐、Leader派单、师傅移动端执行、拍照/不可拍照、处理结果、状态流转、查询、基础权限和日志。报销、采购、报价不做完整模块，只按P1/P2处理。完整HR、薪酬、绩效、仓库、报价、客户门户和AI自动化不进入一期。

## P0 必须上线

| 模块 | 范围 | 验收 |
|---|---|---|
| CS来电/WhatsApp手工建单 | CS录入来源、客户、联系人、电话、地址、问题、附件。 | 3分钟内完成建单。 |
| 工单自动编号 | 按小区类/政府类生成工单编号。 | 编号唯一、可搜索、可追溯。 |
| PM/CM分类 | 选择PM/CM，支持投诉/政府/小区类型标记。 | 管理层可按类型统计。 |
| 区域/Team Leader推荐 | 系统按区域码和地址推荐Leader。 | 推荐结果可手动调整。 |
| Leader派单/改派 | Leader派给师傅，支持改派和拒单后再派。 | 新工单能进入师傅待办。 |
| 师傅移动端接单/拒绝 | 师傅接单或填写拒绝原因。 | 首屏不超过3个核心入口。 |
| 到场记录 | 记录到场时间、位置文字/选点。 | 工单详情可查看到场时间。 |
| 拍照上传/不可拍照豁免 | 上传现场照片；不可拍照客户不强制上传但需说明。 | 照片或豁免原因与工单关联。 |
| 处理结果提交 | 师傅填写处理结果、是否需报价/材料/客户确认。 | 提交后生成流转记录。 |
| 工单状态流转 | 支持主状态和处理中子状态。 | 状态变化有日志。 |
| 工单列表查询 | 时间、状态、负责人、区域、客户、关键词筛选。 | CS/Leader能快速找到目标工单。 |
| 基础权限和操作日志 | 独立账号、角色权限、操作日志。 | 师傅不能查看非授权工单。 |

## P1 一期可选

| 模块 | 可选范围 | 进入条件 |
|---|---|---|
| 工单池主动接单 | 部分工单进入工单池，团队/师傅申请接单，Leader确认。 | P0派单流程稳定后启用。 |
| Outstanding原因看板 | 按等待报价、等待材料、客户确认、施工中、待主管确认分类。 | 客户确认原因枚举。 |
| 超时提醒 | 未接单、待主管确认、等待报价/材料超时提醒。 | 客户确认SLA和提醒对象。 |
| 简版报销关联 | 报销记录选择已完成工单，上传票据，填写金额。 | 不做完整审批/OCR。 |

## P2 只预留字段

| 模块 | 预留字段 | 一期不做 |
|---|---|---|
| 报价 | 报价编号、报价状态、报价负责人、报价附件。 | 不做报价明细、版本控制、PDF自动生成。 |
| 采购/仓库 | PO编号、ML编号、采购到货状态、预计/实际到货日期。 | 不做库存实时校验、扫码入库、盘点。 |
| 客户自助 | 客户自助查询编号、外部Reference No. | 不做客户门户、公开链接查询。 |

## 二期增强

1. 报价单创建、版本控制、客户确认、PDF导出。
2. Material Request、PO流程、ML绑定、到货跟踪深化。
3. 报销OCR识别、审批退回、月度汇总。
4. 客户/供应商公开链接和QR Code。
5. 历史数据批量导入、更多BI报表。

## 三期AI/自动化

1. WhatsApp截图/附件自动抓取。
2. 电话/语音转写和多语言语音输入。
3. 报销、发票、政府PO、位置自动识别。
4. 照片质量校验、设备类型分类。
5. 客户输入编号自助查询进度。
6. 企业知识库与专家推荐。

## 明确暂不做

| 功能 | 原因 |
|---|---|
| 实时师傅地图监控 | 已确认不做，隐私和成本风险高。 |
| 完整HR/薪酬/绩效 | 偏离工单MVP，会议已确认薪酬绩效不上线。 |
| 完整报价系统 | 一期只预留报价编号/状态，不做完整报价流程。 |
| 完整采购/仓库系统 | 仓库已有系统，一期只预留PO/ML/到货状态。 |
| 完整报销审批/OCR | P1最多做简版关联，OCR和完整审批后置。 |
| 完整ERP/CRM集成 | 二期以后评估。 |
| WhatsApp自动抓取 | 一期已确认不做。 |
| 电话语音转写 | 一期已确认不做。 |
| 客户门户 | 一期只预留客户自助查询编号。 |

## 建议POC范围

| 项 | 建议 |
|---|---|
| 试点团队 | 1个区域Team Leader + 3-5名师傅 + 1-2名CS。 |
| 试点数据 | 20-30张真实/模拟工单，包含PM、CM、政府/不可拍照、需报价、需采购。 |
| 试点周期 | 7-10个工作日。 |
| 成功标准 | 师傅能独立完成接单到提交；照片上传成功率≥95%；管理层看板能解释每张Outstanding卡点；权限不串；移动端体验被试点师傅接受。 |

## 一期上线前置条件

1. 客户确认钉钉/宜搭版本、账号上限和存储容量。
2. 客户提供基础数据模板。
3. 客户确认PM/CM流程和编号规则。
4. 客户确认是否强制主管完工确认。
5. 客户确认不可拍照客户清单。
