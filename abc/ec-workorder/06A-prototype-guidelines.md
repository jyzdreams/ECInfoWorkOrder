# HTML原型页面约束

本文件用于下一步生成HTML原型。原型目标是帮助客户确认流程、字段、页面层级和移动端操作，不代表正式生产系统。

## 1. PC管理看板

PC管理看板应体现“工单状态透明”和“Outstanding卡点”，不要做成营销页或炫技大屏。

必须展示的卡片：

| 卡片 | 指标 |
|---|---|
| 今日新增 | 今日新建工单数。 |
| 待处理 | 未派单/未接单数量。 |
| 已接单 | 已被师傅或团队接受数量。 |
| 处理中 | 正在现场或后续处理数量。 |
| 等待材料 | 需采购/到货的数量。 |
| 等待报价 | 需报价的数量。 |
| 待主管确认 | 师傅已提交、Leader未确认数量。 |
| 已完成 | 今日/本周完成数量。 |

必须展示的表格：

| 表格 | 字段 |
|---|---|
| 超时/待跟进工单 | 工单号、客户、地址、状态、Outstanding原因、负责人、停留时长、下一步动作。 |
| 区域/团队概览 | 区域、Team Leader、待处理、处理中、等待材料、等待报价、已完成、超时。 |
| 最新工单动态 | 时间、工单号、操作人、动作、备注。 |

## 2. 工单列表筛选条件

必须支持：

| 筛选 | 说明 |
|---|---|
| 关键词 | 匹配工单号、客户、地址、电话、S/R编号、外部Reference No.。 |
| 创建时间 | 今天、本周、本月、自定义。 |
| 工单状态 | 主状态筛选。 |
| 处理中子状态 | 到场、检查中、等待材料等。 |
| PM/CM | 工单类型。 |
| 区域 | 1P/2P/4P/5P/6P/7P或行政区。 |
| Team Leader | 负责人筛选。 |
| 师傅 | 执行人筛选。 |
| 是否可拍照 | 可拍照、不可拍照。 |
| 是否超时 | 是/否。 |

列表字段建议：工单号、类型、客户、地址、区域、状态、子状态、Leader、师傅、创建时间、到场时间、最近更新、操作按钮。

## 3. 新建工单页面字段

必须字段：

| 字段 | 说明 |
|---|---|
| 来源 | 电话、WhatsApp、系统录入、其他。 |
| 来电时间 | 默认当前时间，可改。 |
| 客户 | 从客户档案选择，可临时新增。 |
| 联系人 | 来电联系人。 |
| 联系电话 | 来电电话。 |
| 地址/屋苑 | 从地址库选择。 |
| 区域 | 根据地址带出，可手动调整。 |
| 工单类型 | PM、CM、投诉、其他。 |
| 业务类型 | 小区类、政府合约类、商业/私营、待确认。 |
| 问题描述 | Call Content。 |
| 附件 | 可上传WhatsApp截图、PDF、照片。 |
| 是否可拍照 | 从客户档案带出。 |
| 推荐Team Leader | 根据区域带出。 |

不应在新建页强制填写：报价明细、PO明细、报销明细、完整项目成本。

## 4. 工单详情页面区块

工单详情必须分成以下区块：

1. 基本信息：工单号、状态、类型、客户、联系人、电话、地址、区域、创建人、创建时间。
2. 派单信息：Team Leader、师傅、派单时间、接单/拒单记录、最近拒单原因。
3. 现场执行：到场时间、开始处理时间、处理结果、提交时间、是否需报价/材料/客户确认。
4. 照片/附件：现场前、现场后、故障细节、材料设备、员工纸、Job Completion、其他。
5. 状态流转：时间线展示创建、派单、接单、到场、上传、提交、确认、归档。
6. 关联信息：报价编号/状态、PO编号、ML编号、到货状态、报销关联入口。
7. 操作区：按角色展示派单、改派、接单、拒绝、到场、上传照片、提交结果、确认完成、取消。

## 5. Team Leader派单工作台

必须包含：

| 区块 | 操作 |
|---|---|
| 待分派工单 | 查看、筛选、分派给师傅、改派Team Leader。 |
| 团队师傅状态 | 显示每个师傅今日待办、处理中、已完成、最近更新时间。 |
| 拒单处理 | 查看拒单原因，重新派单。 |
| 待确认完成 | 查看师傅提交结果和照片，确认完成或退回补充。 |
| P1工单池 | 若启用，审核主动接单申请。 |

## 6. 师傅移动端首屏

首屏最多3个入口：

1. 我的待办。
2. 处理中。
3. 可接工单池（P1启用时显示；未启用则隐藏）。

不要在首屏展示复杂报表、采购、报价、报销、后台配置。

## 7. 师傅移动端工单详情

只显示必要信息：

| 信息/操作 | 说明 |
|---|---|
| 地点 | 客户、地址、区域、地图/导航入口可选。 |
| 联系方式 | 联系人、电话。 |
| 要做什么 | 问题描述、工单类型、特殊备注。 |
| 是否可拍照 | 明确显示“可拍照/不可拍照”。 |
| 关键附件 | 只显示与执行相关的附件。 |
| 操作按钮 | 接单、拒绝、到场、上传照片、提交结果。 |

移动端详情不要显示：成本、完整报价明细、完整采购明细、财务报销明细、后台日志全量内容。

## 8. 关联模块边界

采购、报价、报销只做关联入口：

| 模块 | 原型呈现 |
|---|---|
| 报价 | 显示报价编号、状态、负责人、附件入口。 |
| 采购 | 显示PO编号、ML编号、到货状态、预计/实际到货日期。 |
| 报销 | 显示是否有关联报销、上传票据入口、金额字段。 |

不要做复杂完整模块：报价明细编辑器、采购库存盘点、供应商门户、OCR识别流程、薪酬/绩效。

## 9. UI风格

1. 清晰、业务后台、信息密度适中。
2. 移动端极简，按钮大、步骤少、文本短。
3. PC端适合CS和Leader高频使用，优先列表、筛选、状态、操作。
4. 避免炫技动效、复杂渐变、过度装饰。
5. 不做营销落地页；首屏就是可操作系统。
6. 页面文案使用客户熟悉术语：工单、PM、CM、Team Leader、师傅、到场、拍照、完成、等待材料、等待报价。
