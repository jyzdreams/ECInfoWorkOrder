# 任务拆解

## 1. 基础数据准备

| 项 | 内容 |
|---|---|
| 任务说明 | 整理客户、地址/屋苑、区域码、团队、人员、角色、不可拍照客户、试点历史工单。 |
| 输入 | Daily Service Call Form、区域联系人表、Project Current List、客户/地址Excel。 |
| 输出 | 标准导入模板和首批试点数据。 |
| 验收标准 | 客户/地址去重；繁体中文名称和英文别名齐全；每个地址有区域码和默认Leader。 |
| 依赖关系 | 客户提供现有清单和试点范围。 |
| 优先级 | P0 |

## 2. 数据模型搭建

| 项 | 内容 |
|---|---|
| 任务说明 | 在宜搭搭建工单主表、流转记录、附件、客户、项目、报价、采购、报销、团队配置。 |
| 输入 | 05-data-model.md、字段确认表。 |
| 输出 | 可用的数据表和基础权限。 |
| 验收标准 | 字段可创建、关联可用、必填和枚举符合一期范围。 |
| 依赖关系 | 基础数据准备。 |
| 优先级 | P0 |

## 3. 工单创建与编号

| 项 | 内容 |
|---|---|
| 任务说明 | 实现CS新建工单、普通小区/政府编号生成、附件上传、简单问题已解决。 |
| 输入 | Daily Service Call字段、编号规则。 |
| 输出 | 新建工单页面和编号逻辑。 |
| 验收标准 | 小区类生成日期+序号；政府类支持前缀+ECA+日期+序号；3分钟内可建单。 |
| 依赖关系 | 数据模型。 |
| 优先级 | P0 |

## 4. 工单分派

| 项 | 内容 |
|---|---|
| 任务说明 | 根据区域/团队默认规则推荐Leader，支持手动分派、多团队、改派、拒单再派。 |
| 输入 | 区域/团队配置、PM/CM规则。 |
| 输出 | Team Leader派单工作台。 |
| 验收标准 | 新工单可自动推荐Leader；Leader可派给师傅；拒单后回到待分派。 |
| 依赖关系 | 基础数据、工单创建。 |
| 优先级 | P0 |

## 5. 师傅移动端

| 项 | 内容 |
|---|---|
| 任务说明 | 实现极简移动端：我的待办、接单/拒绝、到场、拍照、处理结果、完成。 |
| 输入 | 移动端页面清单、权限规则。 |
| 输出 | 师傅可用移动端页面。 |
| 验收标准 | 首屏不超过3个核心入口；师傅只看到本人/本组工单；单次提交流程≤2分钟。 |
| 依赖关系 | 工单分派。 |
| 优先级 | P0 |

## 6. 拍照上传

| 项 | 内容 |
|---|---|
| 任务说明 | 支持现场前/后、材料/设备、员工纸、其他照片上传；不可拍照客户豁免。 |
| 输入 | 客户photo_policy、附件分类。 |
| 输出 | 照片附件表和移动端上传组件。 |
| 验收标准 | 可拍照客户提示上传；不可拍照客户不强制但要求填写原因；照片与工单关联。 |
| 依赖关系 | 师傅移动端、客户档案。 |
| 优先级 | P0 |

## 7. 状态流转

| 项 | 内容 |
|---|---|
| 任务说明 | 配置待处理、已接单、处理中、等待报价、等待客户确认、等待材料、待主管确认、已完成、已取消、已归档及子状态。 |
| 输入 | 02-requirements.md状态设计。 |
| 输出 | 状态按钮、状态校验、流转记录。 |
| 验收标准 | 每次状态变化写入日志；管理层可按主状态和子状态筛选。 |
| 依赖关系 | 工单创建、移动端。 |
| 优先级 | P0 |

## 8. 通知提醒

| 项 | 内容 |
|---|---|
| 任务说明 | 钉钉消息/待办通知：新工单、分派、拒单、超时、待确认、等待报价/材料。 |
| 输入 | 通知规则、组织/群配置。 |
| 输出 | 消息模板和触发规则。 |
| 验收标准 | 指定场景能通知到正确角色；超时提醒可配置。 |
| 依赖关系 | 状态流转、团队配置。 |
| 优先级 | P1 |

## 9. 报销关联

| 项 | 内容 |
|---|---|
| 任务说明 | 支持报销记录关联一个或多个已完成工单，上传票据，填写金额和类型。 |
| 输入 | 报销字段、已完成工单。 |
| 输出 | 报销简版页面。 |
| 验收标准 | 报销可关联工单并导出；暂不强制AI识别。 |
| 依赖关系 | 状态流转。 |
| 优先级 | P1可选 |

## 10. 查询筛选

| 项 | 内容 |
|---|---|
| 任务说明 | 实现按时间、状态、负责人、团队、区域、客户、项目编号、关键词查询。 |
| 输入 | 工单主表、客户/项目表。 |
| 输出 | 工单列表筛选器和搜索。 |
| 验收标准 | 输入中文或英文关键词可命中对应字段；不承诺跨语言同义词。 |
| 依赖关系 | 数据模型、基础数据。 |
| 优先级 | P0 |

## 11. 管理看板

| 项 | 内容 |
|---|---|
| 任务说明 | 建立工单总览、区域/团队、Outstanding、工单质量、报价/采购关联看板。 |
| 输入 | 工单状态、子状态、团队数据。 |
| 输出 | 管理层仪表盘。 |
| 验收标准 | 管理层可看到未完成原因、各团队待办和超时工单。 |
| 依赖关系 | 状态流转、查询筛选。 |
| 优先级 | P1 |

## 12. 权限与日志

| 项 | 内容 |
|---|---|
| 任务说明 | 配置角色、数据范围、操作日志、导出留痕。 |
| 输入 | 角色清单、团队/区域配置。 |
| 输出 | 权限矩阵和日志页面。 |
| 验收标准 | 师傅不能看非授权工单；操作可追溯到个人ID；共用账号风险被制度化禁止。 |
| 依赖关系 | 数据模型、钉钉账号。 |
| 优先级 | P0 |

## 13. 测试与验收

| 项 | 内容 |
|---|---|
| 任务说明 | 进行功能测试、权限测试、移动端测试、照片上传测试、真实流程UAT。 |
| 输入 | 验收标准、试点用户、测试工单。 |
| 输出 | 测试报告、问题清单、修复记录。 |
| 验收标准 | P0流程全部通过；POC成功标准达成。 |
| 依赖关系 | 所有P0/P1功能。 |
| 优先级 | P0 |

## 14. 客户演示与反馈修改

| 项 | 内容 |
|---|---|
| 任务说明 | 面向CS、Team Leader、师傅、管理层演示；收集反馈并调整字段/流程/页面。 |
| 输入 | POC环境、试点数据、客户反馈。 |
| 输出 | 演示纪要、变更清单、上线确认。 |
| 验收标准 | 客户确认一期范围、待确认项、二期清单和上线计划。 |
| 依赖关系 | 测试与验收。 |
| 优先级 | P0 |
