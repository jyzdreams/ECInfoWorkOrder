## 不可遗漏信息汇总

| 类别 | 材料证据 | 需求含义 |
|---|---|---|
| 工单 | M2、M4、M5、M6、I1-I4、I14、I16、I20 | 一期核心是来电/WhatsApp建单、PM/CM分类、区域/团队分派、移动端执行、拍照、状态闭环。 |
| 报销 | M3、M4、M5、M6 | 报销需可与工单/工程关联；AI识别可试点，但需人工确认；不做完整薪酬。 |
| 采购 | M1、M4、M6、I29、I31 | 仓库已有系统，初期仅做采购/PO/ML关联和状态查看，不做库存实时盘点。 |
| 报价 | M1、M4、M5、I21-I28、I30 | 报价与工单/项目关联，版本控制重要；一期仅预留或轻量关联，完整报价/标书二期。 |
| 仓库 | M1、M6 | 仓库管理独立，库存人工确认；扫码入库、盘点二期。 |
| 考勤 | M1-M3、M6 | 钉钉打卡可作为平台基础能力；工单系统只用到到场签到/定位，不做薪酬绩效。 |
| 客户自助 | M1、M4、M6 | QR/公开链接/输入编号查进度是远期方向，不进一期。 |
| AI | M1、M2、M5、M6 | AI图文识别报销可试点；语音转写、WhatsApp抓图、照片质量校验、知识库、政府PO识别后置。 |
| 权限/账号 | M1-M3、M5 | 独立账号、禁止共用；近800人超过500通讯录上限，需确认钉钉版本；离职账号数据保留要定策略。 |
| 存储 | M1、M3、M5 | 图片量大、速度和容量是最大技术风险；需压缩、归档、容量采购策略。 |
