# 可研分析

## 1. 是否适合用钉钉/宜搭低代码实现

结论：适合承接一期MVP，但建议先做POC。原因是一期需求以表单、流程、权限、消息通知、移动端数据采集和看板为主，符合钉钉+宜搭低代码能力；但图片量、账号数量、存储、移动端性能和复杂权限是关键风险，需要POC验证。来源：M1、M3、M5、M6。

## 2. 低代码可快速实现功能

| 功能 | 可行性 | 说明 |
|---|---|---|
| 工单表单 | 高 | 宜搭表单可覆盖Daily Service Call Form字段。 |
| 状态流转 | 高 | 通过流程/按钮/字段状态实现。 |
| 移动端填报 | 高 | 钉钉移动端天然适配，但需极简页面。 |
| 附件/照片上传 | 中高 | 能实现，但需容量、压缩、归档策略。 |
| 权限分组 | 中高 | 角色/部门/数据权限可配置，复杂跨团队需设计。 |
| 消息通知 | 中高 | 钉钉群/待办通知可实现，具体接口能力需确认版本。 |
| 筛选/看板 | 中 | 常规报表可做；复杂BI和Outstanding原因拆解需配置数据模型。 |
| 数据导入导出 | 中高 | Excel模板导入导出适合低代码。 |

## 2.1 宜搭实现可行性判断表

| 功能 | 宜搭标准能力是否支持 | 是否需要自定义开发 | 风险等级 | 建议阶段 |
|---|---|---|---|---|
| 工单表单 | 支持 | 否 | 低 | P0 |
| 自动编号 | 基本支持，可通过规则/脚本配置 | 复杂政府前缀可能需要少量配置开发 | 中 | P0 |
| 状态按钮 | 支持 | 否，复杂校验可配置 | 低 | P0 |
| 移动端填报 | 支持 | 否，但需定制极简页面/表单布局 | 中 | P0 |
| 照片上传 | 支持 | 否，容量和大小策略需配置 | 中高 | P0 |
| 图片压缩/归档 | 标准能力有限 | 可能需要自定义压缩、批量归档或外部存储 | 高 | POC后决定，正式归档建议二期 |
| 权限控制 | 支持角色/部门/数据权限 | 跨团队、多角色叠加可能需专项配置 | 中高 | P0 |
| 钉钉群通知 | 支持待办/消息，群推送取决于组织配置 | 复杂群机器人或SLA升级可能需要接口开发 | 中 | P0/P1 |
| 工单看板 | 支持基础报表 | 复杂BI、趋势分析可能需要自定义报表 | 中 | P1 |
| 模糊搜索 | 支持基础字段搜索 | 跨语言同义词搜索需要自定义搜索/词库 | 中 | P0基础，三期增强 |
| OCR识别 | 宜搭本身非完整OCR方案 | 需要AI/OCR接口和人工确认流程 | 中高 | 二期/三期 |
| WhatsApp自动抓取 | 不支持标准直连 | 需要WhatsApp Business API/第三方/截图识别 | 高 | 三期 |
| 客户公开查询 | 可做有限公开表单/链接，需确认安全能力 | 需要外部身份校验、访问码、有效期控制 | 高 | 二期/三期 |

## 3. 可能需要自定义开发的功能

| 功能 | 原因 | 建议阶段 |
|---|---|---|
| WhatsApp自动抓取/电话自动接入 | 需对接WhatsApp Business API、电话系统或第三方，且一期已确认不做。 | 三期 |
| 跨语言同义词搜索 | “battery”搜“电池”当前不支持，需要词库/搜索服务。 | 三期 |
| 大批量图片压缩/归档/外部对象存储 | 宜搭默认附件能力可能受容量和性能限制。 | POC后决定 |
| AI票据/发票OCR | Demo可行但需准确率、人工确认、模型/接口成本。 | 二期/三期 |
| 政府PO/位置自动识别生成工单 | 涉及PDF/OCR/规则解析。 | 三期 |
| 客户自助查询门户 | 公开链接、权限隔离、外部访问安全需单独设计。 | 二期/三期 |
| 完整报价/采购版本控制 | 复杂文档生成、版本对比、确认锁定可能超出简单表单。 | 二期 |

## 4. 依赖钉钉版本、账号数量、存储或接口能力的功能

| 依赖项 | 当前材料信息 | 风险 |
|---|---|---|
| 通讯录人数 | 当前版本上限500，公司近800人。来源：M5 | 必须确认专业版/专属版，否则一线账号无法全覆盖。 |
| 数据保留 | 专业版支持更长保留周期约两年；离职账号当前保留用于查考勤。来源：M5 | 离职账号清理和历史工单归属需策略。 |
| 附件存储 | 月均图片可能达1TB，旧系统存储压力大。来源：M1 | 默认空间可能不够；需压缩、归档或外部存储。 |
| 宜搭许可 | 2025-12-04纪要提到宜搭用于二期定制，初期是否采购需确认。来源：M3 | 若没有宜搭，无法快速搭建工单低代码应用。 |
| 钉钉通知/群推送 | 需要群内推送和待办通知。来源：M2、M6 | 取决于组织配置和接口权限。 |
| 外部公开链接 | 客户/供应商无登录访问特定页面。来源：M4 | 数据安全、访问有效期、权限隔离需确认。 |

## 5. 一期不建议做的功能

| 功能 | 不建议原因 |
|---|---|
| 实时地图监控 | 已确认不做，且有隐私阻力和第三方地图成本。 |
| 完整仓库/WMS | 仓库已有独立系统，一期只需PO/ML关联。 |
| 完整薪酬/绩效 | 非工单闭环，客户内部接受度和合规复杂。 |
| 完整报价/标书系统 | 一期优先PM/CM，报价和标书延后已确认。 |
| 客户自助客服 | 远期规划，需要外部访问和身份校验。 |
| WhatsApp自动化 | 已明确一期不包含。 |
| AI全自动判断 | 当前仍高度依赖人工经验，不宜自动决策。 |

## 6. 主要技术风险

| 风险 | 等级 | 说明 | 应对 |
|---|---|---|---|
| 图片存储量大 | 高 | 会议提及每月上千记录、图片量大，预计月均可达1TB。 | POC测真实单量；移动端压缩；按项目归档；确认容量采购。 |
| 移动端性能 | 高 | 旧系统曾因上传照片、加载工单慢而停用。 | 首屏极简；分页/按本人加载；图片懒加载；限制单次上传大小。 |
| 离职账号数据保留 | 中高 | 当前保留离职账号查历史，未来可能清理inactive。 | 工单归属以人员ID+姓名快照保存；离职后不影响历史。 |
| 权限复杂 | 中高 | CS、CR、Leader、师傅、报价、采购、财务、管理层多角色。 | 一期先按角色+团队+区域做最小权限，避免过度精细。 |
| WhatsApp/电话无法自动接入 | 中 | 一期手动录入，客户可能期待自动化。 | 明确一期边界，三期预留接口字段。 |
| 客户不可拍照限制 | 中 | 监狱等特殊场所不能强制照片。 | 客户档案配置影像限制，流程自动豁免。 |
| 中英文搜索 | 中 | 暂不支持跨语言同义词。 | 一期要求中英别名字段，后续做词库。 |
| 数据导入质量 | 高 | 项目/地址/客户名称需繁体中文和标准区域。 | 导入前清洗模板；设置必填和重复检查；先导入试点区域。 |

## 7. 主要业务风险

| 风险 | 等级 | 说明 | 应对 |
|---|---|---|---|
| PM/CM定义未完全确认 | 高 | 材料已有定义，但具体子类和边界需CS团队确认。 | POC前做流程工作坊，锁定字段和样例。 |
| 五个团队分派规则未确认 | 高 | 纪要提到五个团队默认规则，但具体映射未提供。 | 先按区域/Leader配置表实现可配置。 |
| 师傅是否愿意使用 | 高 | 旧系统失败主因之一是复杂。 | 移动端只保留必要动作，试点收反馈。 |
| 主管是否需要确认完工 | 中高 | M4建议简化，M5又设计待主管确认。 | 一期做可配置，试点默认Team Leader抽核或确认。 |
| 报销是否全员电子化 | 中 | 部分老师傅仍倾向手写。 | 一期支持拍照附件+人工录入，AI/OCR后置。 |
| 政府项目特殊流程 | 中高 | EMSC/政府邮件/不可拍照/编号不同。 | 一期支持政府类型与特殊字段，不强行自动化。 |
| 历史供应商失败导致信任风险 | 高 | 客户曾有供应商未做完并发律师信。 | 先POC、拆分报价、短周期交付可见成果。 |

## 8. 结论

| 判断项 | 结论 |
|---|---|
| 是否建议承接 | 建议承接，但必须以一期MVP+POC方式推进。 |
| 建议一期工期 | 4-6周较稳妥；若只做POC，建议7-10个工作日。M7提到首期交付周期≤30天，可作为商业目标但需控制范围。 |
| 建议一期范围 | P0聚焦来电建单、自动编号、PM/CM、区域/Team Leader推荐、Leader派单、师傅移动端、拍照/不可拍照、处理结果、状态流转、工单列表查询、基础权限日志；P1可选Outstanding/超时/简版报销；P2仅预留报价、PO/ML、客户查询编号字段。 |
| 是否需要先做POC | 需要。尤其验证移动端体验、图片上传、权限、分派规则和看板。 |
| POC成功标准 | 试点一个区域/团队，导入真实客户/地址/人员；完成至少20张真实或模拟工单；移动端上传照片成功率≥95%；师傅单次提交≤2分钟；管理层可看到Outstanding原因；权限不串数据。 |
