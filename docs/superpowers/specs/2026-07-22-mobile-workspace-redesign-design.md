# 移動端工作台重設設計

## 目標

把目前嵌在 PC 原型內、以手機外框呈現的移動頁面，重設為可獨立使用的移動端工作台。師傅與主管共用同一個移動端殼層與 CALL 資料，但各自只看到符合職責的首頁、CALL 清單與處理入口。

## 範圍

本次重設包含：

- 左側導覽新增最末項 `移動端`，放在 `基礎設置` 下方，作為獨立移動工作台入口。
- 從頂部的移動端切換、左側 `移動端`、師傅與主管角色切換，皆進入與角色相符的移動端首頁。
- 重設移動端殼層、首頁、CALL 清單、分派與服務處理頁面之資訊架構和共用組件。
- 保留現有 CALL、狀態流轉、服務報告、師傅清單與通知的模擬資料；移動端操作後需同步反映在原型內的同一份資料狀態。

本次不包含：

- 報價、採購、標書、會計等後台模組的移動端版本。
- 真實檔案上傳、定位、電子簽名或後端 API 串接。
- 更動既有 CALL 流程的狀態定義和權限職責。

## 入口與殼層

### PC 導覽

左側仍以 PC 後台模組為主。`移動端` 固定置於 `基礎設置` 之後、側邊欄最底部的工作區導覽，不混入一般 PC 模組排序。

### 移動工作台

進入移動端後，使用獨立的應用殼層：

- 不顯示 PC 側邊欄、頂部角色列或手機硬體外框。
- 在桌面預覽時，以最大寬度 480px 的純頁面畫布置中呈現，僅保留淺色背景作為預覽環境；在實際手機寬度時填滿視窗。
- 固定頂部列顯示目前角色、團隊或區域、通知入口；固定底部列只提供該角色必要的兩個目的地。
- 原型仍保留角色切換，用於展示不同人員的工作台；成品中的角色由登入身份決定。

## 角色資訊架構

### 師傅

| 目的地 | 內容 | 主要操作 |
| --- | --- | --- |
| 首頁 | 今日待辦、需要優先處理的 CALL、處理進度摘要 | 開啟 CALL、回到最近處理工作 |
| CALL | 依 `待確認`、`待處理`、`處理中`、`已完成` 篩選的個人 CALL | 收到、開始處理、繼續處理、查看完整服務紀錄 |

師傅 CALL 的狀態操作維持明確分段：

1. `待確認` 的「收到」直接更新為 `待處理`，不開啟彈窗。
2. `待處理` 的「開始處理」開啟底部抽屜，完成簽到時間與 Before Service 照片後才可提交為 `處理中`。
3. `處理中` 的「繼續處理」開啟同一個處理抽屜，以 `簽到`、`服務中`、`服務報告` 三段導引，保留即時儲存內容。
4. `服務報告` 區先上傳或加入示例報告，模擬識別後帶入服務報告欄位；完成服務資料、客戶確認及電子簽名後提交，CALL 更新為 `已完成`。
5. `已完成` 的「查看詳情」開啟只讀抽屜，完整呈現服務紀錄、服務報告編號、Before Service 照片、服務中照片、After Service 照片及提交報告照片。

### 主管

| 目的地 | 內容 | 主要操作 |
| --- | --- | --- |
| 首頁 | 待分派、已分派、可接單師傅摘要，以及優先 CALL | 開啟 CALL 清單、查看摘要明細 |
| CALL | 主管範圍內的所有進行中 CALL，優先顯示待分派項目 | 開啟 CALL、分派師傅、查看目前安排與狀態 |

主管在 CALL 清單點擊待分派 CALL 時，從底部開啟分派抽屜。抽屜依序顯示 CALL 摘要、師傅篩選、可選師傅清單、已選師傅與「確認分派」。確認後更新 CALL 的目前負責人、分派時間及狀態，並回到清單。非待分派 CALL 只提供查看安排與處理狀態，不提供重複分派按鈕。

## 共用移動組件

為避免各頁各做一套，移動端只使用下列共用模式：

| 組件 | 用途 | 規則 |
| --- | --- | --- |
| 移動應用列 | 角色識別、頁面標題與通知 | 高度固定，標題以目前頁面為準 |
| 首頁主卡 | 顯示角色最重要的當日工作 | 每個角色僅一張主卡，不作為一般裝飾卡 |
| 摘要指標 | 顯示可追查的工作數量 | 可點擊並開啟相對應的明細或篩選後清單 |
| CALL 卡片 | 顯示編號、客戶、區域、類型、狀態和下一步操作 | 不在卡片堆疊完整後台欄位；詳情由抽屜承載 |
| 分段篩選 | 切換同一清單內的狀態 | 等寬排列，切換後立即重算清單與空狀態 |
| 底部抽屜 | 分派、服務處理和詳情 | 由下往上開啟、可關閉、底部操作固定，內容區可捲動 |
| 雙項底部導覽 | 移動端主要目的地 | 師傅為「首頁 / CALL」，主管為「首頁 / CALL」；名稱相同但頁面內容依角色改變 |

## 資料與互動規則

- PC 與移動端讀取同一份 `tickets`、分派資訊、服務處理資料及通知狀態。
- 篩選只能改變視圖，不可改變 CALL 原始狀態。
- 指標卡、清單、抽屜標題和空狀態都以篩選後的實際資料為準，禁止顯示「待分派 0 張」卻列出其他狀態 CALL 的情況。
- 上傳示例照片或服務報告後，畫面立即顯示已上傳項目與時間；必填項未完成時，提交按鈕提供明確校驗訊息。
- 行動按鈕完成狀態變更後關閉對應抽屜、重新渲染 CALL 清單與首頁摘要，並寫入最近動態。

## 視覺與密度

- 採用現有藍色主色、狀態色、8px 以內圓角和清晰的表單邊界，延續既有組件庫風格。
- 移動端以任務完成為優先：首頁保持短、CALL 清單可一次看多筆、表單僅在抽屜內逐步展開。
- 摘要數字、狀態標籤和主要操作使用色彩區別；普通文字、輔助資料不搶奪視覺層級。
- 禁止在移動畫面中塞入 PC 表格、完整後台導航或未參與當前角色決策的欄位。

## 驗收標準

1. 左側 `移動端` 固定位於 `基礎設置` 下方的側邊欄底部；點擊後開啟獨立移動工作台。
2. 師傅和主管切入移動端後，預設首頁、底部導覽和 CALL 列表均符合各自職責。
3. 師傅可完整走通收到、簽到、上傳 Before Service、服務中記錄、上傳服務報告、客戶簽名、完成 CALL 與查看完整紀錄。
4. 主管可從首頁或 CALL 清單進入待分派 CALL，篩選師傅並確認分派；已分派 CALL 不會顯示錯誤的分派入口。
5. 各列表、指標和空狀態的數量與資料一致，所有移動端抽屜可關閉且不會遺失已儲存的服務資料。
6. 移動端所有重複視覺元素都使用本規格的共用組件模式，不卡在手機硬體外框或 PC 後台導覽中。

## 驗證方式

- 分別選擇師傅與主管角色，從左側 `移動端`、頂部移動端切換進入工作台，確認路由正確。
- 手動走查師傅四個狀態分段、處理抽屜必填校驗、服務報告上傳帶入、完成後服務紀錄。
- 手動走查主管首頁摘要、CALL 篩選、待分派抽屜、師傅篩選與確認分派後的狀態同步。
- 以桌面預覽寬度與窄手機寬度檢查，確保頂部列、卡片、抽屜與底部導覽不重疊或裁切。
