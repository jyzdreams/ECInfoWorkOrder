# 0702 听记原文需求分析报告

> 数据来源：2026年7月2日三次客户沟通听记（纪要+原文）
> 分析日期：2026-07-06

## 1. 客户管理

### 需求描述
- **客户分类简化为两类**："政府"和"私人"，不再细分住宅/商业
- **客户唯一编号**：系统生成，带入后续 PO number 流程
- **政府客户带出合约代码**：选择政府客户时自动带出 EMSD 合约代码（ECA/ECB/ECP/ECS等）
- **合约号码独立字段**：CONTRACT NUMBER 必须独立栏位，不能与其他字段连在一起
- **区域代码**：1-7 及 7A，对应不同服务区域
- **Project Reference 贯穿全程**：CS、报价、service report 共用同一 number 追溯

### 优先级
- 政府/私盘分离、合约号字段、EC前缀选择：**高**
- 区域代码 1-7/7A：**高**
- 客户唯一编号贯穿流程：**高**

---

## 2. 工单编号规则

### 需求描述
- **私人工单**：前缀 `WO` + 系统自动生成的日期序列号
- **政府工单**：前缀根据合约类型手动选择 `ECA`/`ECB`/`ECP`/`ECS` 等，后接系统生成号码
- **前缀固定，后段系统生成**
- **J Number 字段**（政府工单专用）：格式为 `J` + 空格 + 数字（如 "J 123456"），不要点号
- **编号规则 Excel 导入**：政府工单编号规则通过 Excel 配置导入
- **报价单编号独立**：不从 WO 工单派生，从 condition 表批量生成，基于 project code 加 "Q" 前缀

### 优先级
- WO/EC 前缀规则、J Number 字段：**高**
- 报价单编号独立、condition 表批量生成：**高**
- 编号规则 Excel 导入配置：**中**

## 3. 工单状态与流程 ✅ 已实现

### 需求描述
- **6种状态**：待分派、待接单、待处理（新增）、处理中、已完成、已取消
- **移除"待确认"** ✅ 已实现
- **"已关闭"改名为"已完成"** ✅ 已实现

- **工单状态与报价状态并行**
- **区域匹配+负载判断**推荐师傅

### 优先级
- 6种状态重构 ✅ 已实现
- 处理中需打卡+照片触发：**高**
- 状态与报价并行机制：**高**

## 4. 服务报告（Service Report）

### 需求描述
- **SR编号必须由师傅填写**（非系统生成）：来自公司预先打印的纸质报告
- **OCR 自动识别 SR 编号**：从上传的纸质报告左上角自动提取编号（第一期）
- **师傅可编辑已提交的 service report**：增加 edit button，规定时间内可重新编辑
- **必填字段**：SR number、师傅姓名、服务日期、服务类型
- **服务类型分类**：例行检查(routine check)、维修通知(repair)、紧急呼叫(emergency call)
- **服务日期/月份字段**：便于财务按月查阅
- **工程前/后照片**：before/after，UI 分左右板块展示
- **车资字段**：师傅处理 service report 时同步录入车资金额，仅作参考
- **语音输入**：师傅通过语音方式输入排查事项
- **一张工单可对应多个 service report**（2-3个）
- **service report 电子化分两阶段**：
  - 第一阶段：保留纸质流程，拍照后立即上传至钉钉系统
  - 第二阶段：全流程电子化（电子表单→PDF→电子签名）

### 已实现
- SR编号填写（6位数字）✅
- 语音输入 ✅
- 现场照片上传 ✅
- 卡片式分组布局 ✅
- Call代碼（政府客户）✅
- 服务类型多选 ✅
- 工作性质单选 ✅
- 客户满意度 ✅
- 电子签名 ✅
- 是否需要报价开关 + 更换零件 ✅

### 待实现
- OCR 自动识别 SR 编号：**高**
- 师傅编辑已提交的 service report：**高**
- before/after 照片分左右板块：**高**
- 车资字段：**高**
- 一工单多 SR：**中**

## 5. 报价管理

### 需求描述
- **报价单编号独立于工单编号**：避免与工单编号混淆
- **从 condition 表批量生成空白报价单**：不强制绑定工单
- **报价单编号建议**：基于 project code 加 "Q" 前缀生成
- **报价来源多样**：可来自 service report、电话、邮件等，需留 "Tips" 字段录入来源信息
- **报价状态独立管理**：一旦标记"已确定"不可修改
- **报价组可在任意阶段提交报价**：支持多次报价（初次报价、确定性报价）
- **报价单内容结构**：上面固定字段（从客户/工单带出），下面清单（报价人员填写）
- **Project Reference 贯穿**
- **AI 预测报价**（第二期）

### 优先级
- 报价单编号独立、condition表批量生成：**高**
- 报价来源Tips字段、状态独立：**高**
- 多次报价版本、Project Reference 贯穿：**高**
- AI 预测报价：**低**（第二期）
